## Development workflow

1. Create a feature branch from `main`.
2. Make your changes.
3. Run the tests: `cargo test` / `pytest` / etc.
4. Run the linter: `cargo clippy` / `ruff check` / etc.
5. Ensure all existing docs are updated if your change affects the API or behavior.
6. Open a pull request.

## Pull request process

1. Keep PRs focused on a single concern. Split unrelated changes into separate PRs.
2. Write a clear title and description explaining what and why.
3. Link to any related issues.
4. A maintainer will review within a few days.

## Reporting issues

- Check existing issues before filing a duplicate.
- Include: steps to reproduce, expected behavior, actual behavior, environment info.
- Use the provided issue template if one exists.
